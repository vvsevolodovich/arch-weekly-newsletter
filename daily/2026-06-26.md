## [How we built saga rollbacks for Cloudflare Workflows](https://blog.cloudflare.com/rollbacks-for-workflows/) 🤓

Cloudflare's durable execution engine now lets developers attach a compensating action to each `step.do()` call, and on failure it replays rollback handlers in reverse step-start order — turning ad-hoc try/catch cleanup into a first-class saga pattern. The implementation leans on persisted step history to figure out what already ran, reusing existing stubs where possible and reconstructing missing ones during recovery, so rollback inherits the same retries, timeouts, and logging guarantees as the forward path. That matters most for multi-step flows touching external systems — inventory holds, payment authorizations, ticket creation — where partial failures used to mean hand-rolled compensation logic scattered across the codebase. #architecture #distributed #cloud #devops

## [Unlocking the Cloudflare app ecosystem with OAuth for all](https://blog.cloudflare.com/oauth-for-all/) 👨‍💼

Opening self-managed OAuth to every Cloudflare customer required upgrading their Ory Hydra-based OAuth engine from 1.x to 2.x without ever taking production auth offline — a migration that moved 132.5 million existing database rows and inserted 114.7 million more. The two landmines: index operations that would have grabbed exclusive locks on hot tables mid-migration, and Hydra's `SELECT *` queries that would have broken deserialization the instant column layouts shifted. Both got engineered around rather than absorbed as downtime, making this a concrete blueprint for anyone planning a major schema migration on a system that can't pause. #architecture #db #migration #engineering

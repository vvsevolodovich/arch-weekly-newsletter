## [Securing the AI supply chain on GKE: Introducing k8s-aibom for automated AI BOMs](https://cloud.google.com/blog/products/identity-security/introducing-k8s-aibom-on-gke-for-automated-ai-bills-of-materials) 👨‍💼

"Shadow AI" — AI workloads deployed outside formal review channels — is now the default operating reality: k8s-aibom addresses it with a single unprivileged Kubernetes Deployment (no sidecars, no DaemonSets, no eBPF modules) that continuously monitors KServe resources, Deployments, StatefulSets, Jobs, agent frameworks (LangChain, AutoGen, CrewAI), and vector stores (Milvus, Qdrant, pgvector) to produce CycloneDX 1.6 ML-BOM documents. The three-tier confidence model — Declared (explicit config), Inferred (pattern-matched), Unresolved (detected but ambiguous) — creates cryptographically immutable, auditor-traceable evidence chains without modifying existing workloads. Identical cluster inputs produce byte-identical BOMs, making output GitOps-compatible and enabling diff-based AI workload change tracking; exports go to in-cluster AIBOM CRs, GCS buckets, or webhooks. #security #cloud #architecture #devops

## [Zone-Failure-Resilient OpenSearch® at Uber](https://www.uber.com/us/en/blog/zone-failure-resilient/) 🤓

Uber's zone-failure resilience (ZFR) for OpenSearch pairs two mechanisms: forced shard allocation awareness — which refuses to over-allocate shards onto surviving groups when a zone disappears, leaving affected shards unassigned in a yellow state rather than triggering a destructive rebalance — and Uber's isolation-group abstraction over Odin (its container orchestration platform), which guarantees predictable, even node distribution across failure domains. The minimum viable ZFR configuration requires at least 3 shard copies and 5 cluster manager nodes, tolerating a full zone loss plus one additional node failure while retaining quorum. This design cleanly separates the resilience guarantee from the rebalancing policy — a non-obvious but critical split that most distributed search deployments get wrong under failure pressure. #distributed #architecture #engineering #softwareengineering

## [Announcing TypeScript 7.0](https://devblogs.microsoft.com/typescript/announcing-typescript-7-0/) 🤓

TypeScript 7.0 (GA July 8) ships the compiler and language service ported from TypeScript to Go (Project Corsa), preserving identical type-checking semantics while delivering 8×–12× faster full builds — the VS Code codebase drops from 125.7s to 10.6s (11.9×), and Canva's editor sees first-error latency shrink from ~58s to ~4.8s. The key architectural lever beyond native code speed is shared-memory parallelism: the --checkers flag (default: 4) partitions type-checking across Go workers, and --checkers 8 achieves a 16.7× speedup over TypeScript 6 in the VS Code benchmark. The release ships under the standard latest npm tag, making the upgrade path a single package update for most projects — Vue and Svelte require framework-side updates not yet released. #softwareengineering #quality #engineering #development

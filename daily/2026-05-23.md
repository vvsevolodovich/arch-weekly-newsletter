## [Solving the Agent Identity Crisis](https://www.uber.com/us/en/blog/solving-the-agent-identity-crisis/) 🤓

AI agents running as workloads on behalf of humans break the traditional two-category identity model — humans and services — that existing Zero Trust architectures were built around. Uber's solution chains three primitives: an Agent Registry that maps every agent to a Michelangelo ML workload (establishing a verifiable cryptographic principal), a Security Token Service that issues per-hop JWTs scoped to specific destinations with short TTLs to prevent replay attacks, and an MCP Gateway that enforces these policies at every tool-invocation boundary. The STS embeds the full attested actor chain — from the originating user through every intermediate agent — in each token, providing end-to-end traceability without sacrificing developer velocity via a standardized A2A client library; Uber is aligning the design to IETF WIMSE working group drafts to track emerging industry standards. #ai #architecture #softwareengineering #engineering

## [JFrog 2026: AI Governance Fails as Software Supply Chain Attacks Hit Record Highs](https://thenewstack.io/jfrog-ai-supply-chain-security/) 👨‍💼

The software supply chain attack surface expanded well beyond package registries in 2025: JFrog's annual State of the Union report (18.2 billion artifacts analyzed, 1,508 professionals surveyed) found 177K new malicious packages including a 451% npm surge, 495 malicious AI models on Hugging Face, 969 malicious AI agent skills carrying high-impact payloads, and 56 malicious extensions on OpenVSX — all new attack vectors that weren't on the radar two years ago. The "Qix" campaign illustrated the leverage: 25 packages compromised 2.5 million downloads. Despite 97% of organizations claiming AI governance, 53% still pull models from public registries where malicious payloads have been detected, only 40% have malicious package detection active, and just 28% have secrets detection in place. #devops #softwareengineering #ai #engineering

## [How OpenAI Built Its Data Agent](https://blog.bytebytego.com/p/how-openai-built-its-data-agent) 👨‍💼

OpenAI's in-house data agent serves 3,500+ employees across 600+ petabytes and 70,000 internal datasets by pairing GPT-5.5 with an intentionally simple four-layer stack: a single LLM, a context assembly layer, curated tools, and an agent runtime. The reliability edge comes not from model sophistication but from a daily offline pipeline that aggregates table usage logs, human annotations, and Codex-generated enrichments into normalized embeddings — retrieved at query time via RAG over 70,000 table descriptions so the model sees exactly the context it needs. Every answer passes through an Evals grader that verifies both the generated SQL and the resulting data, running continuously as production canaries to catch regressions before they surface. #ai #architecture #engineering #softwareengineering

## [How to Secure Kubernetes in the Age of AI Workloads](https://thenewstack.io/securing-kubernetes-ai-agents/) 👨‍💼

Kubernetes NetworkPolicy's Layer 3/4 scope becomes a fundamental liability when AI agent workloads hit shared GPU nodes: dynamic, ephemeral processes generate bursty, unpredictable egress that IP-level rules cannot inspect at the application layer. Azure Kubernetes Service layers four controls to close this gap — dedicated namespace isolation with NetworkPolicy, admission controllers enforcing node taints, NVIDIA Multi-instance GPU (MIG) partitioning to give each tenant hardware-isolated GPU slices with dedicated compute engines and memory bandwidth, and Defender for Containers' eBPF probes detecting binary drift and anomalous process behavior at runtime. MIG is the decisive primitive: unlike software-only quota enforcement, it eliminates noisy-neighbor GPU contention between co-located AI tenants at the hardware level. #devops #cloud #architecture #engineering

## [Scaling Multi-Node LLM Inference with NVIDIA Dynamo-Grove on AKS](https://blog.aks.azure.com/2026/06/02/dynamo-on-aks-part-4) 🤓

Modern production LLM inference has fragmented into disaggregated prefill and decode phases with distinct GPU, memory, and startup requirements — yet Kubernetes treats them as independent Deployments with no native coupling model. NVIDIA Grove closes this gap with a Kubernetes-native API that describes an entire inference serving pipeline as one object: frontend services, KV-aware routers, prefill workers, decode workers, leader-worker groups, startup dependencies, and scaling constraints, with Grove orchestrating the underlying Kubernetes primitives. This fourth installment of the AKS Dynamo series covers how the teams are converging on topology-aware scheduling and safe rolling updates as the next reliability layer for disaggregated serving pipelines at scale. #ai #distributed #architecture #cloud

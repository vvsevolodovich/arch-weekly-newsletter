## [GitFarm: Git® as a Service for Large-Scale Monorepos](https://www.uber.com/us/en/blog/gitfarm-as-a-service/) 🤓

Uber's GitFarm exposes Git through a high-performance bidirectional gRPC streaming API, eliminating local repository clones entirely and cutting service cold-start times from 10–15 minutes to under 500 milliseconds. The central idea is that treating Git as a service rather than a local filesystem dependency decouples compute from repository state — a pattern that only becomes necessary once a monorepo grows to where full clones are themselves a reliability and resource hazard. Production results confirm the payoff: one service scanning CODEOWNERS files dropped from 70+ cores to 16 (77% reduction) and from 400–600 GB of RAM to 32 GB (92% reduction), with overall client-side resource utilization falling by over 80% across the fleet. #devops #engineering #architecture #softwareengineering

## [What running Kubernetes across millions of clusters taught AWS about zonal failures](https://thenewstack.io/eks-zonal-shift-resilience/) 👨‍💼

The hardest availability failures are "gray zone" failures — an AZ that's degraded but not dead, dropping some traffic while still passing health checks, so standard failover logic never triggers. AWS ARC Zonal Shift for EKS addresses this directly: a manual or automatic control plane that cordons all nodes in the affected AZ, strips them from relevant EndpointSlices, suspends Availability Zone rebalancing, and redirects ingress traffic — without spinning up additional capacity or discarding existing nodes. The zonal autoshift mode closes the operator loop by having AWS use internal network telemetry to detect and trigger shifts autonomously, available at no additional charge. #distributed #cloud #devops #architecture

## [AlphaEvolve is available for everyone](https://cloud.google.com/blog/products/ai-machine-learning/alphaevolve-is-available-for-everyone) 🤓

Google's AlphaEvolve — a Gemini-backed evolutionary code optimization agent — reached GA on July 9, and the production benchmarks from Google's own infrastructure make a strong case for the category: 20% reduction in Spanner write amplification via LSM-tree compaction heuristics, 9% reduction in compiler storage footprint, and 10× lower error rates in quantum circuits. External results span supply chain (80% planning improvement at BASF), ML pipelines (2× throughput and better model quality at Klarna), and IDE performance (15–20% speedup at JetBrains) — all following the same 4-step loop of defining a seed algorithm, writing a scoring function, running evolutionary search over candidate programs, and deploying the winner. The model of framing algorithm engineering as a search problem over coded solution spaces is the architectural bet: any optimization problem with a computable evaluation function is now a near-commodity target for automated improvement. #ai #softwareengineering #engineering #llm

## [Powering the next era of Confidential AI](https://cloud.google.com/blog/products/identity-security/powering-the-next-era-of-confidential-ai) 🤓

For the first time, Apple's Private Cloud Compute privacy commitments extend to third-party infrastructure: Apple Intelligence workloads running on Google Cloud are protected by a hardware TEE stack spanning Intel TDX (CPU-level isolation), NVIDIA Blackwell GPUs with Confidential Computing support, and Google's Titan chip as hardware root of trust. A cryptographically verifiable, append-only ledger tracks every Google Cloud machine in the Apple fleet to prevent supply chain attacks, and all production binaries are published for independent inspection. The architectural claim is that multi-vendor confidential compute can match the trust guarantees Apple's own silicon provides by treating the full stack — firmware, hypervisor, host OS, and inference runtime — as an isolated trusted computing base with no privileged runtime access. #architecture #cloud #ai #engineering

## [ShinyHunters Targets Education Sector with Oracle PeopleSoft Exploit](https://cloud.google.com/blog/topics/threat-intelligence/shinyhunters-targets-education-sector-oracle-exploit) 🤓

CVE-2026-35273 — a CVSS 9.8 unauthenticated remote code execution bug in Oracle PeopleSoft's Environment Management Hub — was exploited from May 27 to June 9, 2026, with victim data appearing on ShinyHunters' leak site the day before Oracle published its advisory. Attackers deployed MeshCentral C2 agents disguised as Azure services, parsed `/etc/hosts` to fan out via SSH credential spraying, and compressed exfiltrated data with zstd — all automated within hours of initial access. Over 100 organizations were hit, 68% of them in higher education; immediate mitigation is disabling the PSEMHUB application or blocking external HTTP access to `/PSEMHUB/*` and `/PSIGW/HttpListeningConnector`. #engineering #softwareengineering #devops

## [Quick: An internal hosting platform for the AI era](https://shopify.engineering/quick) 👨‍💼

Shopify's Quick started as the simplest possible internal hosting: HTML files in a GCS bucket, a wildcard NGINX config mapping `*.quick.shopify.io` to folder paths via GCS Fuse, and Google Identity-Aware Proxy ensuring every request is a verified Shopify employee. In under a year it hosts 50,000+ sites used by more than half the company — AI code generation created demand for internal hosting that outpaced every alternative tool. The architecture grew only when forced: an LLM proxy with server-side API keys, optional CloudSQL+Node.js backends, BigQuery data connectors, and WebSocket support, each added when multiple teams independently needed it. #engineering #devops #cloud #softwareengineering

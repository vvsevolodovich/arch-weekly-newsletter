## [Multi-agentic Software Development is a Distributed Systems Problem (AGI can't save you from it)](https://kirancodes.me/posts/log-distributed-llms.html) 🤓

Multi-agent coding pipelines aren't just complicated — they're formally subject to the FLP impossibility theorem, because message delivery to an LLM is asynchronous and the agent decides when (or whether) to act on it, which is exactly the assumption FLP needs to rule out guaranteed consensus in bounded time. The piece treats agent misinterpretation as the LLM-world equivalent of a crash fault, then argues the fix isn't smarter models but external validation — tests, static analysis, formal verification — that converts silent misinterpretations into detectable crash failures, the same workaround distributed systems use to live with FLP in practice. It's a pointed rebuttal to the "just wait for better models" school of agentic coordination. #distributed #ai #llm #architecture

## [Defend against frontier cyber models: Cloudflare's architecture as customer zero](https://blog.cloudflare.com/frontier-model-defense/) 👨‍💼

Cloudflare runs its own production security stack against its own infrastructure as "customer zero" for defending against AI-driven attacks, and lays out the layers: Bot Management scores every request for automation signals before a frontier model can finish mapping an attack surface, Zero Trust Network Access replaces implicit internal trust with per-request identity checks on every employee-facing tool, and a new Require Access Protection control blocks newly deployed or misconfigured apps from being externally reachable before a policy exists. The throughline is that AI-accelerated reconnaissance and exploitation make defense-in-depth architecture — not patch speed — the deciding factor in whether a vulnerability becomes a breach. #security #architecture #cloud #ai

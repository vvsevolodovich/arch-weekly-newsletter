## [Your Worker can now have its own cache in front of it](https://blog.cloudflare.com/workers-cache/) 👨‍💼

Cloudflare's new Workers Cache places a two-tier regional cache directly in front of Worker entrypoints, solving the problem that Workers previously had to implement their own cache-hit logic via the Cache API. The two-tier topology mirrors standard CDN design: on a lower-tier miss the request promotes to an upper aggregating tier, and only on a combined miss does the Worker execute — populating both layers on the response path. One practical consequence is that the first cold request from anywhere in the world warms the upper tier, so any subsequent Cloudflare data center globally can serve that key without invoking the Worker at all. Deployment requires a single wrangler.toml entry plus Cache-Control headers on Worker responses, with no changes to application code. #cloud #architecture #devops

## [Run log analytics for a fraction of the cost with the new engine for Amazon OpenSearch Service](https://aws.amazon.com/blogs/big-data/run-log-analytics-for-a-fraction-of-the-cost-with-the-new-engine-for-amazon-opensearch-service/) 🤓

AWS's new purpose-built log analytics engine for OpenSearch Service stores data in Apache Parquet and writes searchable fields to a parallel inverted index — then uses Apache Calcite to route each query to Apache DataFusion (columnar analytics) or Lucene (text-search predicates) at parse time, so a single query can engage both engines without data movement. Benchmarks show 70% lower storage, 2× faster analytical queries, 2× higher ingest throughput, and 4× overall price-performance over the standard OpenSearch engine at no additional charge. The design ends the historical forced choice between OLAP column stores and full-text search for observability workloads — the same service now handles both access patterns from one data copy. Available in preview with opt-in during cluster creation. #db #architecture #softwareengineering

## [Beyond Static Prompts: Building Scale-Proof, Polymorphic Multi-Agent Systems with Google's ADK](https://cloud.google.com/blog/topics/developers-practitioners/beyond-static-prompts-with-google-adk) 👨‍💼

Pre-loading domain schemas into agent system prompts — a widespread multi-agent anti-pattern — forces 20,000+ tokens of dead context into every request and causes attention diffusion, where the model silently borrows field definitions from schemas irrelevant to the current task. Externalizing schemas to a Cloud Storage metadata registry and fetching them lazily via a 200-token discovery prompt cuts active context by ~99%, enables zero-downtime schema changes through JSON file replacement, and routes validation hooks to Cloud Run endpoints instead of inline prompt instructions. A four-phase lifecycle (discovery → metadata resolution → assembly → finalization) enforces deterministic field validation before any enterprise API call, eliminating the silent handoff failures common in nondeterministic agent pipelines. #ai #architecture #softwareengineering

## [Expanding Private Cloud Compute](https://security.apple.com/blog/expanding-pcc/) 🤓

Running Apple Intelligence inference outside Apple's own data centers for the first time required stacking three independent hardware trust layers on a single server: NVIDIA Confidential Computing on Blackwell GPUs, Intel TDX on the host CPUs, and Google's Titan chip as an additional root of trust — so that no single component failure or compromise exposes inference data. Apple maintains a cryptographically verifiable, append-only ledger of every Google Cloud server in the PCC fleet, and attestation is rooted in at least two separate roots of trust from independent vendors, eliminating the single-authority trust assumption of conventional cloud deployments. The four core PCC requirements — stateless computation, no privileged runtime access, non-targetability, and verifiable transparency — transfer unchanged to the third-party deployment, making this the first confidential AI inference architecture to enforce those properties at global scale outside the hardware owner's own facilities. Rolling out through summer 2026 as a preview on NVIDIA Blackwell. #cloud #architecture #security #engineering

## [Amazon S3 Annotations: Attach Rich, Queryable Context Directly to Your Objects](https://aws.amazon.com/blogs/aws/amazon-s3-annotations-attach-rich-queryable-context-directly-to-your-objects/) 👨‍💼

S3's metadata model previously capped objects at 2 KB of user-defined metadata and 10 tags; Annotations replaces this with up to 1,000 structured entries per object and a combined 1 GB capacity in JSON, XML, or YAML — updatable independently of the object, which makes them practical for AI-generated summaries, compliance classifications, lineage records, and audit trails that evolve after the object is written. Enabling annotation tables on a bucket automatically materializes all annotations into a fully managed Apache Iceberg table queryable with Amazon Athena, Redshift, or any Iceberg-compatible engine, collapsing the separate metadata-store pattern many teams maintained alongside raw S3. A natural-language discovery path for AI agents is provided through the S3 Tables MCP server, making annotation-indexed buckets addressable in agentic retrieval pipelines without custom tooling. #cloud #db #ai #architecture

## [How We Built Cloudflare's Unified Data Platform](https://blog.cloudflare.com/our-unified-data-platform/) 🤓

Cloudflare's analytics estate spanned Postgres, ClickHouse, and fragmented SaaS sources until Town Lake unified them under a single query surface built on Apache Trino, Apache Iceberg, R2 object storage, and DataHub as the metadata catalog — a single Trino query can now join live ClickHouse rows with historical Iceberg snapshots without moving data between systems, collapsing the ETL pipelines that maintained analytical replicas. Billing workloads account for 53% of all platform queries (91,760 queries measured from 324 Cloudflare employees); governance is default-closed — every newly onboarded dataset remains inaccessible until Skimmer's automated PII classification, AI-based analysis, and human review cycle completes before access is granted. Skipper, the natural-language SQL agent built on top, queries DataHub to discover relevant tables and schemas, generates SQL, submits it to Trino, and returns tables or charts directly — a pattern applicable to any team building analyst-facing access layers over a multi-source lakehouse. #db #architecture #ai #distributed

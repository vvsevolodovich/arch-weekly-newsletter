## [Scaling MCP Adoption: Cloudflare's Enterprise Reference Architecture](https://blog.cloudflare.com/enterprise-mcp/) 👨‍💼

Cloudflare's enterprise MCP reference architecture tackles the token-cost problem at its root: Code Mode collapses any MCP server's full tool catalog into just two portal tools (`portal_codemode_search` and `portal_codemode_execute`), reducing a 2,500-endpoint API surface from over 1.17 million tokens to roughly 1,000—a 99.9% reduction with a fixed footprint that doesn't grow as APIs expand. The architecture delegates remote MCP server management to a single platform team, routes all inter-service calls through AI Gateway for per-user spend tracking and budget enforcement, and uses Cloudflare Access JWT as the OAuth provider. For platform teams standardizing on MCP at enterprise scale, this is a production-complete blueprint: remote hosting, cost controls, identity, audit logging, and progressive tool disclosure in one reference design. #architecture #cloud #ai #engineering

## [How OpenAI Built a Secure Windows Sandbox for Codex Agents](https://www.infoq.com/news/2026/06/codex-windows-sandbox-design/) 🤓

OpenAI built a custom Windows sandbox for Codex after finding existing OS isolation mechanisms insufficient for autonomous coding agents. The sandbox creates dedicated low-privilege local accounts (CodexSandboxOffline and CodexSandboxOnline), runs agent commands under restricted tokens, and enforces a synthetic SID called `sandbox-write` that grants write access only to the current workspace; Git metadata directories remain ACL-protected even from within the workspace, preventing agents from modifying repository history. Parts of the sandbox implementation have been open-sourced to improve enterprise trust and enable security auditing of the isolation boundaries. #architecture #engineering #ai #softwareengineering

## [Stragglers, Not Failures: How Adaptive Hedged Requests Reduce p99 Latency by 74 Percent](https://www.infoq.com/articles/adaptive-hedged-requests-p99-latency/) 🤓

In fan-out microservice architectures, the dominant cause of high p99 latency is stragglers—slow-completing requests, not failures—because one straggler in a fan-out blocks the entire composite response. The mechanism uses DDSketch for constant-memory (O(1)) per-host quantile estimation with ±1% relative error and ~35ns per-request overhead, combined with a token-bucket budget that caps the hedge rate at a configurable traffic percentage and automatically stops hedging when all services are slow, preventing the load-doubling spiral during genuine outages. The result is a 74% reduction in p99 latency with zero call-site configuration changes, with a reference implementation available as an open-source Go library at github.com/bhope/hedge. #distributed #architecture #engineering #softwareengineering

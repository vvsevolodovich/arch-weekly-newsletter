## [Temporal Primer — Building Long-Running Systems](https://arpitbhayani.me/blogs/temporal-primer/) 🍼

Temporal is a durable execution engine that records every workflow event — activity scheduled, timer fired, signal arrived — as an immutable entry in an event history persisted to Cassandra, PostgreSQL, or MySQL, then replays that history on worker restart so a process survives network partitions and VM reboots without any custom state management code. Workflow code must be strictly deterministic (no random values, no wall-clock reads, no direct I/O); all side effects live in Activities, which carry independent retry policies. For agentic systems that chain dozens of tool calls over minutes or hours, this durability model solves exactly the failure class that cron + queue + DB hacks break under. #distributed #architecture #softwareengineering #llm

## [I/O '26 news for agent developers on Google Cloud](https://cloud.google.com/blog/topics/developers-practitioners/io26-news-for-agent-developers-on-google-cloud) 👨‍💼

Google's I/O agent platform update formalizes a four-tier abstraction stack — no-code Agent Studio, Managed Agents API (configuration-first, ephemeral sandbox per execution), Antigravity 2.0 (multi-agent orchestration desktop), and code-first ADK 2.0 — all unified by the A2A protocol so agents at any tier can be called as sub-agents by others. ADK 2.0's Python SDK v2.0.0 adds three explicit sub-agent execution modes: `chat` (full user interaction), `task` (auto-return to parent after clarification), and `single-turn` (parallel, no user interaction). The Agent Platform governance layer — Identity, Gateway, Security, Registry — ships alongside, turning one-off agent experiments into auditable infrastructure backed by Cloud OAuth and per-agent sandbox isolation. #ai #llm #cloud #architecture

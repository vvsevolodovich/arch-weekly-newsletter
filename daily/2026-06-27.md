## [Argo CD 3.5 Tightens Supply Chain Security with Internal mTLS and Source Integrity](https://www.infoq.com/news/2026/06/argocd-supply-chain-security/) 🤓

Argo CD 3.5 closes a gap that's been hiding in plain sight: traffic between the repo-server and other internal components was unencrypted even in clusters that enforce mTLS at the ingress layer. The repo-server now requires client certificates from every connecting component, generating self-signed certs in memory when no custom PKI is configured, so internal health checks stay verifiable without depending on the filesystem. Alongside that, a new Source Integrity feature lets operators require valid Git commit signatures before a sync runs — set via `sourceIntegrity.required: true` on the Application spec or `argocd app set --source-integrity-required` — replacing the deprecated `signatureKeys` field. For GitOps shops worried about a compromised commit silently rolling out to production, this is a direct, low-effort hardening step. #devops #architecture #security #engineering

## [Slack Outlines Four-Phase Journey to a Multi-Cloud AI Serving Platform](https://www.infoq.com/news/2026/06/slack-multicloud/) 👨‍💼

Slack's AI serving stack went from a self-managed Amazon SageMaker deployment — manually forecasting capacity and fighting for scarce A100/H100 allocations — to a full multi-cloud architecture spanning AWS Bedrock and Google Cloud Vertex AI. The pivotal phase wasn't adding a second cloud; it was building a hybrid capacity model that routes latency-sensitive traffic to Bedrock's Provisioned Throughput endpoints while letting bursty background jobs spill into On-Demand, absorbing workload swings Slack says can hit 10x between peak and off-peak. The provider-agnostic serving layer that resulted — secretless auth, normalized APIs, unified observability — delivered a ~10% quality bump on complex reasoning tasks and cut short-prompt latency by roughly 67%. #cloud #architecture #ai #engineering

## [AI Is Moving up the Software Lifecycle: from Code Review to PRD Governance](https://www.infoq.com/news/2026/06/ai-prd-code-review-governance/) 👨‍💼

Code review was the easy part — Uber, DoorDash, and Cloudflare are now pushing AI review further left, into PRDs and design docs, before a single line of code exists. Uber runs a first-pass AI filter on product requirement documents while keeping engineers as final validators, and DoorDash's internal code reviewer is explicitly tuned for fewer, higher-signal comments after teams complained about noisy automated feedback. Cloudflare's framing is the sharpest: deciding what the system should *not* flag is as important as deciding what it should, since false positives are what kill trust in an AI reviewer fastest. The pattern across all three is the same — AI as a first-pass filter at each lifecycle checkpoint, with humans retaining sign-off. #ai #softwareengineering #engineering #quality

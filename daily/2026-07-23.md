## [In-House LLM Serving at Netflix](https://netflixtechblog.com/in-house-llm-serving-at-netflix-a5a8e799ea2c) 🤓

Netflix chose vLLM over TensorRT-LLM in summer 2025 and integrated it directly into NVIDIA Triton via the vLLM backend, routing LLMs through the same Model Scoring Service (MSS) that already handles XGBoost, TensorFlow, and PyTorch models — a deliberate bet on one unified serving path over separate LLM-specific infrastructure. The architecture decouples model artifacts from frontend code: a Java control plane sits above Triton handling versioning, autoscaling, and multi-region rollout independently of which engine runs below. vLLM won over TensorRT-LLM on four criteria: loading custom model architectures without a compilation pipeline, extensibility hooks for custom decoding logic, better debuggability, and practitioner familiarity inside the ML org. The workload mix — embedding generation, prefill-only ranking inference, autoregressive decoding, and custom per-step constraint models — was the forcing function; by summer 2025 open-source engines had closed the performance gap with specialized stacks on every axis that mattered. #llm #architecture #engineering #ai

## [Why we cannot wait for better post-quantum signature algorithms](https://blog.cloudflare.com/ml-dsa-will-have-to-do/) 👨‍💼

ML-DSA signatures run 2,420–4,595 bytes against ECDSA's 64 bytes — 38–72× larger on the wire — and that size difference alone causes ~5% of TLS connections to fail on middlebox-heavy networks while slowing the rest. Cloudflare's case for migrating to ML-DSA now rather than waiting for nine new NIST post-quantum signature candidates still in evaluation is purely temporal: those candidates won't be standardized until 2030 at earliest, which is after federal mandates require post-quantum readiness. The practical migration architecture is asymmetric — ML-DSA replaces ECDSA for authentication while ML-KEM (already covering more than half of human-initiated web traffic through Cloudflare as of late 2025) handles key agreement — because authentication signatures have no backward-compatible upgrade path in TLS 1.3. #security #architecture #engineering #softwareengineering

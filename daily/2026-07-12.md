## [C4N Network and Storage-Optimized VMs](https://cloud.google.com/blog/products/compute/c4n-network-and-storage-optimized-vms) 🤓

The defining architectural choice in C4N is a Titanium offload chip that moves all NIC and block-storage controller work to dedicated silicon, freeing 5th-gen Intel Xeon Emerald Rapids cores for pure compute. Network throughput reaches 400 Gbps per VM and 95 million packets per second (224% faster than comparable Intel setups), while storage peaks at 25 GiB/s and near-1M IOPS with Hyperdisk Extreme — all included at no premium add-on cost. MySQL QPS improves 45% for disk-resident workloads, and a ClipperDB + Spark analytics stack achieved 3× lower cost per query and 11× faster query time in customer benchmarks. #cloud #architecture #engineering #softwareengineering

## [Announcing etcd v3.7.0](https://kubernetes.io/blog/2026/07/08/announcing-etcd-3.7/) 👨‍💼

etcd v3.7.0's headline addition is RangeStream: instead of buffering an entire range response in memory before replying, the server now streams result sets back to clients in chunks, directly eliminating the OOM failure mode that surfaces when Kubernetes clusters grow large configmap counts or high-cardinality CRD data volumes. The release also completes the removal of the legacy v2store — etcd now boots entirely from v3store, closing a multi-year maintenance debt — and bumps its two core dependencies to bbolt v1.5.0 and raft v3.7.0. Operators running clusters with large watch cardinality or high-volume range queries should treat this as a priority upgrade. #distributed #devops #architecture #softwareengineering

## [Improving Smart Tiered Cache for public cloud regions](https://blog.cloudflare.com/smart-tiered-cache-for-public-clouds) 👨‍💼

Cloudflare's Smart Tiered Cache has historically failed to pick the right upper-tier CDN node for origins hosted on AWS, GCP, Azure, or Oracle Cloud because those providers use anycast or regional unicast networking, making latency probing from Cloudflare PoPs unreliable for geo-inferring origin location. The fix is a single cloud-region hint (e.g., us-east-1) that instructs Smart Tiered Cache to select a geographically close primary upper-tier node plus a resilience fallback in a separate location, collapsing cold-path latency for cloud-hosted origins. Teams using Cloudflare in front of cloud-native backends and seeing poor cache-hit ratios despite correct cache headers should check their upper-tier topology — this is likely the root cause. #cloud #devops #architecture

## [The post-quantum EO is an important milestone. Now it's time to get to work](https://blog.cloudflare.com/post-quantum-eo-2026/) 👨‍💼

Executive Order 14409, signed June 22, 2026, mandates migrating federal systems to NIST's post-quantum cryptography standards — but it says nothing about what happens when those standards need to change again. Cloudflare's counter-argument: crypto agility, the ability to swap cryptographic algorithms without re-architecting everything around them, is the actual deliverable, not compliance with any single algorithm list. Treating algorithm choice as a pluggable layer rather than something hardcoded into protocol and certificate logic is what separates a multi-year re-architecture from a config change the next time NIST revises guidance. A useful framing for any team starting PQC migration now: design for the next swap, not just this one. #security #architecture #cloud #engineering

## [Architecting AI-powered resilience framework on AWS](https://noise.getoto.net/2026/06/22/architecting-ai-powered-resilience-framework-on-aws/) 👨‍💼

AWS's five-layer framework turns resilience testing from a weeks-long manual scavenger hunt into an automated pipeline: Discovery, Test Generation, Experimentation, and Gap Analysis layers feed AWS Resilience Hub as the orchestration hub, with a fifth Continuous Validation layer wiring the loop into CI/CD. The clever part is discovery — a custom agent on Amazon Bedrock AgentCore, running in an isolated microVM for sessions up to 8 hours, scans code repositories directly for hardcoded dependencies, connection strings, timeouts, and retry logic that architecture diagrams and runbooks never capture. AWS Fault Injection Service then generates chaos experiments targeted at what discovery actually found, instead of generic failure injection. A concrete blueprint for teams who know they should be chaos-testing more but lack the specialized expertise to design meaningful experiments by hand. #devops #cloud #architecture #ai

## [Telemetry that matters: Designing sustainable, high-impact observability pipelines](https://www.cncf.io/blog/2026/06/22/telemetry-that-matters-designing-sustainable-high-impact-observability-pipelines/) 👨‍💼

Roughly 50% of collected metrics are never queried or acted upon, yet every one still gets stored, indexed, and billed — a waste that compounds storage costs, alert noise, and the carbon footprint of the platform collecting it. The fix isn't instrumenting less; it's pushing cardinality control and sampling decisions into the pipeline layer itself, so platform teams can adapt to data explosions without forcing application teams to rewrite and redeploy code. Smart sampling beats pure random sampling here, since random drops can silently erase the rare error signals that matter most mid-incident. The recommended rollout: start with zero-code auto-instrumentation for a baseline, then layer in manual instrumentation only where deep context earns its cost. #devops #cloud #engineering #architecture

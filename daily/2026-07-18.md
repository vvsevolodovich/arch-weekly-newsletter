## [Introducing Apache Spark 4.2](https://www.databricks.com/blog/introducing-apache-spark-42) 🤓

Spark 4.2 ships 1,900 commits across four architectural bets that tighten Spark's claim on AI-native analytics: Metric Views bring a native semantic layer to Spark SQL — dimensions and measures become first-class engine objects so aggregation semantics survive intact across dashboards, applications, and AI agents instead of diverging per team; built-in GEOGRAPHY and GEOMETRY types following the OGC Simple Feature Access spec eliminate the need for external spatial extensions with Parquet, WKT/WKB, and SRID support out of the box; AutoCDC in Spark Declarative Pipelines declaratively handles SCD Type 1 updates via the new CHANGES SQL clause built on a standardized Data Source V2 change stream API; and Arrow-first Python execution through Spark Connect makes PySpark substantially cheaper to invoke from agents and external services. The release is available today in Databricks Runtime 19 Beta, with the 1,900-commit scope making it the largest Spark 4.x minor release to date. #distributed #db #softwareengineering #development

## [HAMi Becomes a CNCF Incubating Project](https://www.cncf.io/blog/2026/07/15/hami-becomes-a-cncf-incubating-project/) 👨‍💼

HAMi (Heterogeneous AI Computing Virtualization Middleware) moves GPU and AI accelerator scheduling on Kubernetes from whole-device allocation — which leaves expensive accelerators fragmented and underused — to fine-grained slicing by memory, core count, or device unit, with hard runtime isolation enforced between co-resident workloads. The architecture is three components: a Mutating Webhook that rewrites scheduler fields transparently without touching application manifests, a Scheduler Extender that applies binpack, spread, and topology-aware placement policies, and vendor-specific Device Plugins covering NVIDIA, Huawei Ascend, Cambricon, Hygon DCU, Moore Threads, AWS Neuron, and seven other accelerator families. The project was accepted into CNCF Sandbox in August 2024, reached Incubating in July 2026 backed by 550+ contributing organizations and five published production case studies spanning education, cloud platforms, and enterprise deployments. #cloud #devops #architecture #engineering

## [Operating OpenTelemetry at Scale with OpAMP](https://www.cncf.io/blog/2026/07/13/operating-opentelemetry-at-scale-with-opamp/) 👨‍💼

At 100,000-Collector scale, every Collector maintaining a direct WebSocket connection to a central management platform becomes an immediate bottleneck — OpAMP (Open Agent Management Protocol) breaks this by inserting an OpAMP Gateway Extension that runs inside a Collector and acts as a multiplexer, so 1,000 edge Collectors aggregate into one gateway connection instead of 1,000 direct connections to the backend. The gateway extension, targeting launch around KubeCon Europe 2026, lets a central platform remotely configure agents, push updates, monitor health, and collect status information in a standardized way across heterogeneous Collector fleets without per-agent bespoke management logic. This is the operational scaling layer that was visibly absent as OpenTelemetry matured from a tracing project to the vendor-neutral default for metrics, logs, and traces in production environments. #devops #cloud #architecture #softwareengineering

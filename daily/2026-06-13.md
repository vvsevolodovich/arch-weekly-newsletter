## [Simplifying Data and Product Integrations with a Data Abstraction Layer](https://www.uber.com/us/en/blog/data-abstraction-layer/) 👨‍💼

Uber's Data Abstraction Layer (DAL) is an RPC service exposing a single `FetchData` endpoint where callers specify table name, schema, time range, and filters — replacing direct dataset coupling that previously made schema evolution require weeks of cross-team coordination. Built initially for Uber Ads' advertiser reporting, the DAL cuts multi-week report development to under two days by letting producers evolve internal data models without forcing downstream consumers to change. The pattern is domain-agnostic: stable API contracts over dataset topology are worth the indirection layer whenever producer and consumer teams iterate at different speeds. #architecture #db #softwareengineering #engineering

## [Introducing Durable Functions in PostgreSQL](https://techcommunity.microsoft.com/blog/adforpostgresql/introducing-durable-functions-in-postgresql/4526821) 🤓

Microsoft's pg_durable makes PostgreSQL an orchestration engine: workflows are defined as graphs of SQL steps, each checkpointed after execution, so database crashes and restarts resume from the last completed step with no external state reconstruction. The architecture is intentionally minimal — a Postgres extension plus a background worker, no external control plane or message queue — supporting fan-out parallelism via a `&` operator, per-step retries, and scheduled execution entirely inside the database process. This directly challenges Temporal and similar external orchestrators by co-locating execution with data and leveraging Postgres' native durability guarantees; the project reached 1.7K GitHub stars within days of its June 9 open-source launch. #db #architecture #engineering #softwareengineering

## [When History Fails You, Borrow from Geography](https://medium.com/airbnb-engineering/when-history-fails-you-borrow-from-geography-915a72b91b5c) 🤓

When market shocks make historical time-series data unreliable, Airbnb borrows forecast signal from geographically similar corridors that are structurally ahead in the recovery curve — a technique Harrison Katz calls sequential geographic recovery propagation. The method uses Bayesian compositional hierarchical modeling to weight priors from analogous markets and propagate them through a similarity graph, outperforming pure history-based models precisely when historical data diverges most sharply from current conditions. Airbnb's global footprint is the enabling structural advantage: enough market diversity to find informative geographic analogues for almost any corridor facing disrupted history. #ai #softwareengineering #engineering

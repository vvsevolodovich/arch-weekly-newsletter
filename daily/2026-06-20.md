## [Ten years of ClickHouse in open source](https://clickhouse.com/blog/open-source-10) 🤓

ClickHouse started in 2009 as a bet that real-time analytical reports could be generated directly from non-aggregated, constantly-arriving data — a hypothesis it took three years to prove before going into production at Yandex.Metrica in 2012. Open-sourced on June 15, 2016, it's now the most popular open-source analytical database with more than 2,000 contributors, each credited in the changelog and even queryable from `system.contributors`. The headline number is throughput: where contemporary systems topped out around a hundred thousand rows per second, ClickHouse's column-oriented MergeTree engine pushes hundreds of millions of rows per second on the same class of hardware. A decade-long case study in winning by going narrow — one workload, ruthlessly optimized — instead of general-purpose. #db #architecture #engineering

## [Build your own vulnerability harness](https://blog.cloudflare.com/build-your-own-vulnerability-harness/) 👨‍💼

Cloudflare's security team frames the harness, not the underlying LLM, as the real engineering artifact: they started from a roughly 450-line security-audit prompt run against a single repo, then layered on orchestration that manages state across multi-stage scans, uses adversarial review to squash false positives, and routes around context-window limits. That architecture is model-agnostic by design — they swap in whichever frontier model is currently strongest without touching the harness — and the pipeline now discovers, validates, deduplicates, and opens a working pull request in roughly 14 hours end to end. It's a concrete blueprint for treating AI agent infrastructure as a durable system to engineer rather than a thin wrapper around a model call. #security #ai #devops #engineering

## [Datasette Apps: Host custom HTML applications inside Datasette](https://simonwillison.net/2026/Jun/18/datasette-apps/) 👨‍💼

Datasette Apps sandboxes HTML+JS apps inside an iframe with an injected CSP that blocks any outbound network call, then bridges `datasette.query()` and `datasette.storedQuery()` calls back to Datasette's own JSON API via postMessage — so every query still runs under the current actor's permissions. The result is Claude Artifacts with a real relational database behind it instead of throwaway state: sandboxed, untrusted compute talking to governed SQL through one narrow, auditable channel. It's a clean, reusable pattern for anyone wiring LLM-generated or user-submitted UI to a database without handing it direct access. #architecture #db #ai #engineering

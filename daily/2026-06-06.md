## [Enabling Evolutionary Database Development: database branching with Lakebase, continued](https://www.databricks.com/blog/enabling-evolutionary-database-development-database-branching-lakebase-part-2) 🤓

Lakebase's copy-on-write storage architecture reduces database branching to an O(1) metadata operation: a 1TB Postgres database branches in the same one second as a 1MB database, with zero bytes copied at creation and only modified pages consuming additional storage. The architecture is practical because Lakebase separates compute (the Postgres engine) from storage (shared S3-backed pages), making per-developer, per-PR, and per-experiment Postgres instances as routine as code branches. Part 2 of the Evolutionary Database Development series covers nine practices, including CI-gated schema migrations with the diff posted inline to PRs, DBA responsibilities shifting from change approval to policy administration, and AI agents creating database branches alongside human developers. #db #engineering #architecture #development

## [Your AI bill is out of control. Cloudflare can fix it now.](https://blog.cloudflare.com/ai-gateway-spend-limits/) 👨‍💼

Cloudflare AI Gateway now enforces spend budgets in dollars rather than tokens — eliminating the mapping problem between token counts and actual cost across heterogeneous models and providers. Limits can be scoped to any combination of model, provider, or custom metadata dimensions (user, team, application) with fixed or rolling time windows, and Dynamic Routes redirect to a fallback model rather than hard-blocking production traffic when a limit triggers. The identity layer validates the Cloudflare Access JWT in a Gateway Worker, maps user email to an anonymous UUID using D1 (persistent) and KV (read cache), and attaches only that UUID to AI Gateway metadata — giving per-user cost attribution without exposing identities to model providers. #cloud #ai #engineering #architecture

## [Coding Is No Longer the Constraint: Scaling Developer Experience to Teams and Agents at Spotify](https://engineering.atspotify.com/2026/6/code-with-claude-coding-is-no-longer-the-constraint) 👨‍💼

Spotify's decade-long investment in Fleet Management, Backstage, and engineering standardization turns out to be the exact infrastructure that makes AI agents productive — 99%+ of engineers now use AI coding tools weekly, 94% report increased productivity, and PR frequency is up 76%, with 2.5 million automated maintenance PRs merged so far. The decisive insight is that the same constraints that make teams effective (standardized environments, CI test gating, scoped agent permissions) make agents effective too: none of this works without mature feedback loops already in place. As coding velocity increases the constraint shifts from writing code to making decisions, and Spotify is now rethinking planning and prioritization as the bottleneck moves from implementation to judgment. #engineering #ai #softwareengineering #architecture

## [Agentjacking: A Fake Bug Report Hijacks Your AI Coding Agent](https://tenetsecurity.ai/blog/agentjacking-coding-agents-with-fake-sentry-errors/) 🤓

AI coding agents' attack surface now includes every upstream data source surfaced through MCP-connected services: Tenet Security weaponized Sentry's open event ingestion — requiring only a public write-only DSN key and an HTTP POST — to inject attacker-controlled instructions into error events that Claude Code, Cursor, and Codex receive through the Sentry MCP server and execute as trusted remediation guidance. The attack bypasses EDR, WAF, IAM, VPN, and Cloudflare entirely because every action in the chain is technically authorized — Tenet terms this the "Authorized Intent Chain" — and achieved an 85% success rate across over 100 real-world targets. Sentry acknowledged the issue on June 3, 2026 but declined to fix it at the platform level, leaving the only viable defensive layer inside the agent runtime itself, at the moment the agent decides to act. #ai #llm #engineering #softwareengineering

## [How Flat Is Replacing Fat in AWS Data Center Networks](https://www.amazon.science/blog/how-flat-is-replacing-fat-in-aws-data-center-networks) 🤓

A result from early-1990s graph theory — that quasi-random connections between switches produce the most efficient, resilient topologies — has taken 30 years to reach hyperscale cloud infrastructure. Amazon's Resilient Network Graphs (RNG) replaces the fat-tree hierarchy with a flat, quasi-random ToR-to-ToR mesh wired through passive optical ShuffleBoxes, cutting router count by 69%, boosting throughput by 33%, and reducing network power consumption by 40%. The first production deployment went live in Dublin at the end of 2024; by April 2026 RNG became the default architecture for most new AWS data centers globally, representing the largest known production deployment of expander-graph network topology in any cloud provider. #cloud #architecture #engineering #distributed

## [Over 400 Arch Linux AUR Packages Compromised to Push Rootkit and Infostealer (Atomic Arch)](https://www.bleepingcomputer.com/news/security/over-400-arch-linux-packages-compromised-to-push-rootkit-infostealer/) 🤓

Attackers systematically claimed ownership of orphaned AUR packages using Arch Linux's standard adoption process, then rewrote PKGBUILD scripts to silently install a Rust credential stealer — targeting SSH keys, browser sessions, GitHub and npm tokens, HashiCorp Vault material, Slack and Discord data, and cloud IAM credentials — exfiltrating everything to a Tor C2. On machines where root is achieved, a companion eBPF rootkit hides the malicious process from ps, top, and /proc inspection, making standard incident detection blind. A second wave on June 12 pushed the total from 408 packages to over 1,500; the structural vulnerability is the AUR trust model itself, where the orphaned-package adoption process grants anonymous maintainers unconditional PKGBUILD execution rights with no security review gate. #devops #engineering #softwareengineering
